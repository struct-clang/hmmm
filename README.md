# Hmmm
Hmmm (Hide My MAX Messages) - проект для автономного шифрования сообщений в мессенджерах.
В первую очередь проект создан для шифрования сообщений из мессенджера MAX, но также может использоватся для работы с любыми мессенджерами поддерживающими отправку файлов

## Подготовка к использованию
Установите Python 3+ и зависимости: `cryptography lxml`

## Использование

### 1. Генерация пар ключей и обмен публчиными ключами.
Сначала, Вы и Боб запускаете `gen.py`.
Этот код сгенерирует вашу связку ключей.
Скрипт запросит ваш ник, вы можете указать любой.


После генерации, вы получите `alice_private.xml` и `alice_public.xml` в директории `keys/`, где `alice` - ваш ник который вы ввели. 
Поделитесь своим `alice_public.xml` со теми, кто должен получать сообщения от вас.

Если вы получили этот публчиный ключ, поместите его в директорию `keys`

> [!CAUTION]
НИКОГДА не делитесь приватным ключoм!

### 2. Проверка публичных ключей для избежания MITM.
После генерации пар ключей используйте команду `sha256sum keys/alice_public.xml` (`alice` - ваш ник) - она выдаст хэш вашего публичного ключа. Отправьте его по провренному каналу связи, лучше всего - распечтаать и отправить по реальной почте, или использовать любой другой доверенный канал связи.


Чтобы проверить публчиный ключ Боба, выполните `sha256sum keys/bob_public.xml` (`bob` - ник вашего получателя) и сверьте хэш с тем, что вы получили от Боба.

> [!NOTE]
Это не обязательный этап, но он помогает избежать MITM аттаки через подделку публичного ключа. 


### 3. Начало общения.
Для того чтобы начать общение выполните команду `send.py --keys path/to/keys/directory --text path/to/text.txt --for bob`: `path/to/keys/directory` - путь к директории с ключами, `path/to/text.txt` - путь к тексту для шифрования, `bob` - ник получателя.
На выходе вы получите файл `message_bob.xml` - вы можете безопасно отправить его в любой недоверенный источник.


Для расшифровки используйте `see.py --encrypted path/to/message.xml --keys path/to/keys/directory`: `path/to/message.xml` - путь к полученному сообщению, `path/to/keys/directory` - путь к директории с ключами. В терминале вы получите исходное сообщение.


### 4. Советы

> [!TIP]
Для удобства использования, пишите собщения больше, вкладывая в них больше смысла. Вы потратите меньше времени на шифрование и расшифровку от Боба соответсвенно.


> [!TIP]
Не используйте сам недоверенный канал связи для проверки публчиных ключей.
